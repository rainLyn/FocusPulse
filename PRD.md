## 1. 产品概述

### 1.1 一句话定义

一个「专注计时 + 时间统计」的轻量 iOS 工具——你选择分类开始专注，结束后自动生成当天的统计饼图。

### 1.2 核心价值

| 问题 | FocusPulse 的解法 |
|------|------------------|
| 学了一天但不知道时间花在哪 | 每次专注打上分类标签，统计自动归集 |
| 计时器开了但总是忘了关 | 计时状态常驻展示，随时可以结束 |
| 看不到长期趋势，缺乏持续动力 | 每日统计图表，清晰直观 |
| 统计工具太复杂，配置成本高 | 三秒上手：选分类 → 开始 → 结束 → 看图 |

### 1.3 设计哲学

```
「反馈循环」是一切的核心
   记录 → 可视化 → 反思 → 调整
        ↕         ↕
  操作成本极低    反馈即时可见
```

每次交互都不能超过两步。每多一次点击，长期坚持的概率就下降一个数量级。

## 2. 用户画像与场景

### 2.1 典型用户

| 画像 | 描述 |
|------|------|
| 考研/高考备考生 | 每天多个科目轮换学习，需要精确掌握各科时间分配 |
| 自由职业者 | 跨项目工作，需要记录不同客户的时间投入 |
| 习惯养成者 | 想培养「每天 X 小时专注」的习惯，需要图表反馈 |

### 2.2 核心场景

**场景 A：开始一次专注**
1. 用户打开 App → 首页显示分类列表
2. 点击「高等数学」→ 点击「开始专注」
3. 系统：启动计时器，全屏计时界面
4. 用户：锁屏 / 切到其他 App

**场景 B：结束一次专注**
1. 用户回到 App → 计时界面显示当前专注时长
2. 点击「结束」→ 数据持久化
3. 返回首页 → 今日饼图自动刷新

**场景 C：查看统计**
1. 用户打开 App → 底部 Tab「统计」
2. 饼图：今日各科目时长分布
3. 向下滚动 → 时段明细列表

### 2.3 边界场景与异常处理

| 场景 | 行为 |
|------|------|
| 专注中 App 被系统杀死 | 下次启动时检测到未结束会话，弹窗提示「检测到上次专注未正常结束，已自动记录 X 分钟？[确认]/[丢弃]」 |
| 跨午夜专注 | 会话按实际时间段归入开始日期，不影响统计准确性 |
| 分类为空时点击开始 | 引导用户先创建第一个分类 |
| 同时开始两个专注 | 禁止——先结束当前专注才能开始新的 |
| 极短的专注（< 30 秒） | 视为误触，不记录；但仍然触发一次震动反馈提示 |
| 删除已有时长记录的分类 | 二次确认弹窗「该分类下有 X 条专注记录，删除后数据将隐藏（不删除原始记录）」 |

## 3. 核心功能规格

### 3.1 分类管理

```
┌─────────────────────────────────────┐
│           分类管理子系统               │
├─────────────────────────────────────┤
│ 核心职责: CRUD + 排序 + 颜色管理      │
│ 输入约束: 名称 1-12 字符，不可重名     │
└─────────────────────────────────────┘
```

**功能列表**

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| C-01 | 创建分类 | P0 | 输入名称 + 选择颜色（预设 12 色） |
| C-02 | 编辑分类 | P0 | 修改名称和颜色 |
| C-03 | 删除分类 | P0 | 软删除——归档而非物理删除 |
| C-04 | 排序调整 | P1 | 通过拖拽排序，越常用越靠前 |
| C-05 | 预设模板 | P2 | 首次启动时预置「高数」「英语」「专业课」「休息」四个分类 |

**数据约束**

```
名称: 1-12 字符，trim 后不可为空，不可与其他分类同名（忽略大小写）
颜色: 12 色预设色板，不可自定义色值（防止选择瘫痪）
上限: 最多 20 个分类（防止泥团）
```

### 3.2 专注计时器

```
┌─────────────────────────────────────┐
│           专注计时器引擎               │
│                                      │
│  状态机:                             │
│                                      │
│  [空闲] ──start──> [专注中]           │
│    ^                    |            │
│    |                   end           │
│    └───── [已结束] <────┘            │
│                                      │
│  核心承诺: 不丢秒、不漂移             │
└─────────────────────────────────────┘
```

**功能列表**

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| T-01 | 选择分类后开始专注 | P0 | 必须在分类页选中一个分类才能开始 |
| T-02 | 实时计时显示 | P0 | hh:mm:ss 格式，每秒刷新 |
| T-03 | 结束专注 | P0 | 记录结束时间，计算总时长 |
| T-04 | 后台计时精度 | P0 | 使用 `Date` 时间戳差而非 `Timer` 累计（Timer 在后台会被暂停） |
| T-05 | 中断恢复 | P0 | 见 2.3 异常处理 |

**计时精度实现策略**

```
不用 Timer 累加秒数——App 进入后台后 Timer 会暂停。
正确做法:
  startTime = Date.now  (记录开始绝对时间)
  前台 UI 每秒: elapsed = Date.now - startTime
  结束: duration = endTime - startTime
```

## 4. 数据模型

### 4.1 实体关系

```
┌──────────────────┐       ┌──────────────────────┐
│   Category        │       │   FocusSession        │
├──────────────────┤       ├──────────────────────┤
│  id: UUID         │──1:N──│  id: UUID             │
│  name: String      │       │  categoryId: UUID     │
│  colorHex: String  │       │  startTime: Date      │
│  iconName: String  │       │  endTime: Date?       │
│  sortOrder: Int    │       │  durationSeconds: Int │
│  isArchived: Bool  │       │  createdAt: Date      │
│  createdAt: Date   │       │  updatedAt: Date      │
└──────────────────┘       └──────────────────────┘
        1                            N
        │                            │
        │                            │
        │       ┌────────────────────┘
        │       │
        │  ┌──────────────────┐
        │  │   DailySummary   │  (物化视图，每日聚合)
        │  ├──────────────────┤
        └──│  date: Date      │
           │  categoryId: UUID│
           │  totalSeconds: Int│
           │  sessionCount: Int│
           └──────────────────┘
```

### 4.2 表结构详述

```swift
// ═══════════════════════════════════════
// Category - 专注分类
// ═══════════════════════════════════════
struct Category: Identifiable, Codable {
    let id: UUID
    var name: String          // 分类名称，唯一约束
    var colorHex: String      // 6位 hex，如 "4A90D9"
    var iconName: String      // SF Symbol 名称，如 "book.fill"
    var sortOrder: Int        // 排序权重，升序
    var isArchived: Bool      // 软删除标记
    let createdAt: Date
}

// ═══════════════════════════════════════
// FocusSession - 每条专注记录
// ═══════════════════════════════════════
struct FocusSession: Identifiable, Codable {
    let id: UUID
    let categoryId: UUID      // 外键 → Category
    let startTime: Date       // 专注开始绝对时间
    var endTime: Date?        // nil = 专注进行中/异常中断
    var durationSeconds: Int  // 缓存计算值，0 = 异常中断
    let createdAt: Date
    var updatedAt: Date
}

// ═══════════════════════════════════════
// DailySummary - 每日聚合（物化视图）
// 每日结束后惰性计算，减少实时查询压力
// ═══════════════════════════════════════
struct DailySummary: Identifiable, Codable {
    let id: UUID
    let date: Date            // 日期（不含时间）
    let categoryId: UUID      // 外键 → Category
    var totalSeconds: Int     // 该日该分类总秒数
    var sessionCount: Int     // 该日该分类专注次数
    var updatedAt: Date
}

// ═══════════════════════════════════════
// AppPreference - 应用偏好
// ═══════════════════════════════════════
struct AppPreference: Codable {
    var hasCompletedOnboarding: Bool
    var lastActiveDate: Date?     // 上次打开日期（用于检测跨天）
    var totalLifetimeSeconds: Int // 累计总专注时长（缓存）
    var longestStreak: Int        // 最长连续专注天数（缓存）
}
```

### 4.3 数据一致性保障

| 规则 | 实现 |
|------|------|
| `FocusSession.durationSeconds` = `endTime - startTime` | 写入时由模型层自动计算，UI 层不参与 |
| 删除分类时 => 其下的 FocusSession 不删 | 外键关系为「弱引用」，session 的 categoryId 保留，但 UI 层标记为「已删除分类」|
| `DailySummary` 的刷新时机 | 每次 FocusSession 写入时异步更新，最多延迟 500ms |
| 跨天检测 | App 进入前台时检查 `lastActiveDate`，若跨天则触发前一天的 `DailySummary` 补算 |

### 4.4 存储选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 持久化主存储 | SwiftData (或 CoreData) | 关系型、Swift 原生、支持 iCloud 同步（未来） |
| 用户偏好 | UserDefaults / @AppStorage | KV 小数据，无需数据库 |
| 临时状态 | ViewModel @Published / @State | 内存态，不落地 |

## 5. 技术架构

### 5.1 分层架构

```
┌─────────────────────────────────────────────────────┐
│                     Presentation Layer               │
│  SwiftUI Views · Charts · Animations                  │
│  └─ MVVM: Views ←→ ViewModels (@ObservableObject)  │
├─────────────────────────────────────────────────────┤
│                     Domain Layer                     │
│  TimerEngine · AggregationService · CategoryService  │
│  └─ 纯 Swift，无 UIKit/AppKit 依赖                   │
├─────────────────────────────────────────────────────┤
│                     Data Layer                       │
│  SwiftData Store · Cache · UserDefaults              │
│  └─ Repository Pattern 封装所有读写                   │
└─────────────────────────────────────────────────────┘
```

### 5.2 依赖方向

```
View → ViewModel → Service → Repository → SwiftData
```

**关键约束：**
- **View 决不能直接访问 Repository 或 SwiftData**
- **Service 层是纯业务逻辑，不能 import SwiftUI**
- **Repository 是数据层唯一入口**，SwiftData 替换不影响上层

### 5.3 核心模块说明

```swift
// ═══════════════════════════════════════
// TimerEngine - 计时器引擎
// 纯逻辑，无 UI 依赖，可独立单元测试
// ═══════════════════════════════════════
final class TimerEngine: ObservableObject {
    enum State { case idle, running(Date), ended(FocusSession) }

    @Published private(set) var state: State = .idle
    @Published private(set) var elapsedSeconds: Int = 0

    // 唯一公开方法
    func start(categoryId: UUID) { ... }
    func stop() -> FocusSession { ... }
}

// ═══════════════════════════════════════
// TimerEngine 不被 ViewModel 持有，
// 而是作为一个 @StateObject 注入。
// 这保证引擎生命周期与视图树绑定。
// ═══════════════════════════════════════
```

```swift
// ═══════════════════════════════════════
// DailySummaryRepository
// Repository 封装所有数据查询
// ViewModel 通过它获取数据，不直接操作 SwiftData
// ═══════════════════════════════════════
actor DailySummaryRepository {
    func summary(for date: Date) async throws -> [DailySummary]
    func refresh(date: Date) async throws
}
```

### 5.4 关键技术决策

| 计时精度 | 时间戳差而非 Timer 累加 | App 后台 Timer 暂停，时间戳差不受影响 |
| 饼图 | Swift Charts (Apple 原生) | iOS 16+ 原生支持，声明式 API，性能好 |
| 数据刷新 | Combine Publisher + 自动订阅 | FocusSession 写入后自动通知相关 ViewModel 刷新 |
| 并发安全 | Repository 设计为 Swift Actor | 防止多线程数据竞争 |

- View 文件只放布局代码，逻辑全部抽到 ViewModel

## 6. 用户界面设计

### 6.1 设计原则

```
简洁 ≠ 简陋    —— 每个像素都有目的
清晰 ≠ 啰嗦    —— 用颜色和空间代替文字说明
优雅 ≠ 华丽    —— 克制是最高级的审美
```

1. **内容优先**：核心信息（时间、统计数据）直接展示，不藏进汉堡菜单
2. **单层导航**：最多两级页面深度，杜绝导航迷宫
3. **即时反馈**：每次操作（开始/结束）都有明确视觉反馈（触感 Haptic + 动画）
4. **自适应布局**：单手可操作，核心按钮在屏幕下半部分

### 6.2 页面架构

```
Tab 1: 专注 (首页)          Tab 2: 统计
┌─────────────────┐       ┌─────────────────┐
│ 专注分类列表     │       │ 日期切换器       │
│ [高数] [英语]    │       │   ← 2026-5-30 →  │
│ [专业课] [休息]  │       │  (点击弹出DatePicker)│
│ [+ 添加分类]    │       │ ┌───────────┐   │
│                 │       │ │  饼图     │   │
│ [开始专注]      │       │ │           │   │
│                 │       │ └───────────┘   │
│ ────────       │       │ ─────────────── │
│ 今日概览       │       │ 时段明细列表    │
│ 饼图缩略+总计  │       │ 14:00-15:30 高数 │
└─────────────────┘       │ 16:00-17:00 英语 │
                          │ ─────────────── │
                          │ 专注日历        │
                          │ ┌──┬──┬──┬──┐  │
                          │ │日│一│二│三│  │
                          │ ├──┼──┼──┼──┤  │
                          │ │  │  │  │ 1│  │
                          │ │  │  │  │2h│  │
                          │ ├──┼──┼──┼──┤  │
                          │ │ 2│ 3│ 4│ 5│  │
                          │ │1h│30│  │  │  │
                          │ └──┴──┴──┴──┘  │
                          │ （点击日跳转）  │
                          │ [清除数据]      │
                          └─────────────────┘
```

### 6.3 交互细节

| 操作 | 交互反馈 |
|------|---------|
| 点击分类卡片 | 卡片弹性缩放 (0.97x → 1.0x, 300ms spring) |
| 点击开始专注 | 卡片翻转动画进入计时界面，同时触发 Haptic Feedback (medium impact) |
| 结束专注 | 进度环收拢动画 + Haptic Notification (success) |
| 饼图切换日期 | 左/右滑手势 + 饼图旋转入场；点击日期弹出 DatePicker |
| 空状态 | 插画 + 引导文字 + 按钮「去创建一个分类」|
| 点击日历日期 | 日历格子选中高亮 + 饼图/明细切换到该日 |
| 清除数据 | 红色按钮 + DatePicker 弹层 + 二次确认弹窗 |

### 6.4 视觉规范

| 属性 | 值 |
|------|-----|
| 圆角半径 | 12pt (卡片) / 8pt (按钮) / 6pt (标签) |
| 卡片阴影 | y: 2, blur: 8, color: black @ 8% |
| 字体 | SF Pro (系统默认) |
| 字号层级 | 34(大标题) / 20(标题) / 15(正文) / 13(辅助) |
| 色板 | 12 色固定色板，饱和度 60-70%，明度 70-80% |
| 间距 | 8 点网格系统，所有间距为 8 的倍数 |
| 留白策略 | 顶部 20pt，左右 16pt，卡片间距 12pt |

### 6.5 空状态与加载态

| 页面 | 空状态 | 加载态 |
|------|--------|--------|
| 首页 (无分类) | 插画 + 「还没有分类，先创建一个吧」+ 按钮 | 骨架屏（3 个灰色卡片）|
| 首页 (有分类无记录) | 饼图区域显示「今日暂无记录」| 饼图 skeleton loading |
| 统计页 (无记录) | 饼图区域显示「今天还没有专注记录」| 灰色 placeholder |
| 专注中页面 | N/A（必有内容） | 计时器直接显示 00:00 |

## 8. 验收标准

### 8.1 功能验收

| ID | 标准 | 测试方式 |
|----|------|---------|
| F-01 | 创建分类后立即出现在首页列表 | 手动 + UI 测试 |
| F-02 | 选择分类后点击开始，计时器从 0 开始增长 | 手动 |
| F-03 | App 进入后台再返回，计时器时间正确（非暂停重算）| 手动 |
| F-04 | 点击结束，数据保存到 SwiftData，饼图刷新 | 单元测试 + 手动 |
| F-05 | 跨天后首页饼图自动清空（显示新的一天）| 手动（模拟时间）|
| F-06 | 删除分类时有二次确认，历史记录不丢失 | 手动 |
| F-07 | 专注时长 < 30s 不记录 | 单元测试 |
| F-08 | 点击统计页日期弹出 DatePicker，选择后跳转到正确日期 | 手动 |
| F-09 | 日历格子按专注时长着色，点击跳转到对应日统计 | 手动 |
| F-10 | 清除数据选择日期后二次确认，确认后数据正确删除 | 手动 + 数据验证 |

### 8.2 性能验收

| 指标 | 目标 |
|------|------|
| App 启动时间 (冷启动) | < 1.5s |
| 饼图渲染时间 | < 200ms (1000 条 session 记录) |
| 内存峰值 | < 50MB |
| 包体积增量 | < 15MB |

### 8.3 体验验收

| 指标 | 目标 |
|------|------|
| 完成「开始一次专注」的操作步骤 | ≤ 2 次点击（选分类 + 开始） |
| 首页信息密度 | 一眼能看到所有核心信息 |
| 空状态覆盖 | 所有列表/图表页面均有空状态占位 |

### 8.4 不发版的信号

```
❌ 任何「开始计时 → 结束」流程导致数据丢失
❌ 饼图数据与实际记录不一致
❌ App 在专注中崩溃且没有恢复机制
```
